## 0. 게임 시작 / 진행 흐름

### 0-1. 인원별 분배 (총 48장)

| 인원 | 손패 (각자) | 바닥 | 더미 | 검산 |
|---|---|---|---|---|
| **2인** (맞고) | **10장** | **8장** | 20장 | 10×2 + 8 + 20 = 48 ✓ |
| **3인** | **7장** | **6장** | 21장 | 7×3 + 6 + 21 = 48 ✓ |
| **4~5인** (광팔이 후 실제 3인) | 7장 | 6장 | 21장 | (3인 게임과 동일) |

### 0-2. 시작 점수 (난다 기준)

| 인원 | 점수 |
|---|---|
| 2인 (맞고) | **7점** |
| 3인 / 광팔이 후 | **3점** |

### 0-3. 분배 방식

선부터 반시계방향:
- **3인**: 4장씩 한 바퀴 → 바닥 3장 → 다시 3장씩 한 바퀴 → 바닥 3장 (총 손패 7장 / 바닥 6장)
- **2인**: 5장씩 한 바퀴 → 바닥 4장 → 다시 5장씩 한 바퀴 → 바닥 4장 (총 손패 10장 / 바닥 8장)

✅ **구현됨**: `dealNewGame`이 인원수에 따라 분기. 본인 시각화는 `lib/dealingPattern.ts`로 4-3-3 stagger.

## 1. 점수 계산 — 베이스

### 1-1. 광 (3장 이상)

| 보유 | 점수 | 비고 |
|---|---|---|
| 3광 | 3점 | 비광 미포함 |
| 비삼광 (3광 중 12월 비광 포함) | **2점** | 비광이 끼면 -1점 |
| 4광 | 4점 | 비광 포함이어도 4점 |
| 5광 | 15점 | 광박 자동 적용 (×2 → 30점) |

### 1-2. 끗 (열끗) (5장 이상)

| 보유 | 점수 |
|---|---|
| 5장 | 1점 |
| 6장 | 2점 |
| 7장 | 3점 + **멍따 ×2** (별도 보너스) |
| 8장 | 4점 + 멍따 |
| 추가 1장당 +1점 |

**고도리** (1·2·8월 새 3장): **+5점** 별도 보너스.

### 1-3. 띠 (5장 이상)

| 보유 | 점수 |
|---|---|
| 5장 | 1점 |
| 6장 | 2점 |
| 추가 1장당 +1점 |

**홍단 / 청단 / 초단 (쿠사)** 각 3장 모음 시 각 **+3점** 별도 보너스.

### 1-4. 피 (10장 가치 이상)

| 보유 | 점수 |
|---|---|
| 10장 | 1점 |
| 11장 | 2점 |
| 추가 1장당 +1점 |

**쌍피**: 한 장 = 2장 가치 (11월 쌍피, 12월 비쌍피).
**조커** (옵션, 미구현): 쌍피 또는 쓰리피로 미리 약속.

### 1-5. 9월 열끗 쌍피 변환 (옵션) ✅ 구현됨
9월 열끗 카드를 끗 자리 또는 쌍피 자리로 자유 토글 (멀티 한정, 본인 보유 시 SettingsModal에서 토글).
`calculateScore({ nineYeolAsSsangPi })` 옵션으로 점수 계산.

### 1-6. 매칭 처리 ⭐ 정통 룰 (Image #20 표) ✅ 구현됨

손패 1장을 내고 더미를 1장 뒤집는 매 턴의 매칭 처리:

| # | 상황 | 결과 (용어) | 설명 |
|---|---|---|---|
| 1 | 바닥 0 + 손 1 + 더미 같은 월 | **쪽** | 바닥엔 없었지만 내가 내고 뒤집은 게 짝. **상대 피 1장씩 뺏어옴** |
| 2 | 바닥 1 + 손 1 + 더미 다른 월 | 그냥 먹기 | 일반 매칭. 손패 + 바닥 1장 = 2장 collected |
| 3 | 바닥 1 + 손 1 + 더미 같은 월 | **뻑 (설사)** | 2장 먹으려다 3장이 겹침. 못 가져오고 바닥에 3장 묶어둠 (stuck) |
| 4 | 바닥 2 + 손 1 + 더미 다른 월 | 그냥 먹기 | 바닥 2장 중 1장만 짝 맞춰 가져옴 (1장은 바닥에 남음). **종류 다르면 사용자 선택** ⭐ |
| 5 | 바닥 2 + 손 1 + 더미 같은 월 | **따닥** | 1장 먹고 더미로 나머지 1장까지 = **총 4장**. **상대 피 1장씩 뺏어옴** |
| 6 | 바닥 3 + 손 1 | **뻑 회수** (먹기) | 묶여있던 3장 + 손패 1장 = **총 4장 한 번에**. 자뻑이면 **2장씩**, 일반 회수면 **1장씩** 뺏어옴 |

**선택 모달 노출 조건** (rules-final.md §1-6 case 4):
- 바닥에 같은 월 카드 2장 + 손패가 같은 월
- 두 바닥 카드의 종류가 **다를 때만** 선택 모달 (피+광 등)
- 같은 종류 (피 2장 등)면 자동 첫 번째 — 결과 동일
- 종류 비교: `kind` + `ddiKind` + `isSsangPi` + `isGoDori` + `isBigwang`
- 손패 단계 / 더미 단계 각각 검사 — 둘 다 해당하면 모달 2번
- 모달 시점: **Phase 3 (더미 비행) 후** 표시 → 선택 후 Phase 4 (점수판 비행)

> ⚠️ **싹쓸이 발생 시**: 위 매칭 후 바닥이 비어버리면 `sweep` flag + 피 1장 추가 (마지막 턴 제외).

> **server protocol**: `game:action play-card { cardId, targetAfterHand?, targetAfterDraw? }` →
> 종류 다른 2장 매칭 시 `{ ok: false, needsSelection: { stage, candidates } }` 응답.
> 클라가 모달 띄우고 선택 후 `targetAfterHand`/`targetAfterDraw` 포함하여 재요청.

## 2. 박 (Bak) — 점수 ×2

피박/광박/멍박은 **승자에게 곱셈** (각각 ×2). 누적 시 ×4, ×8.

| 박 | 발동 조건 | 비고 |
|---|---|---|
| **피박** | 승자가 피로 점수 났음(10장+) AND 패자 피 **1~5장** | 패자 피 **0장 = 면제** ⭐ |
| **광박** | 승자가 광으로 점수 났음(3광+) AND 패자 광 **0장** | |
| **멍박** | 승자가 끗으로 점수 났음(5장+) AND 패자 끗 **0장** | |
| **고박 / 독박** | 고 부른 사람이 다른 사람에게 졌을 때 | 점수 ×2, 본인이 다른 패자 몫까지 부담 |

### 2-1. 멍박 vs 멍따 (별개 개념) ⭐
- **멍박**: 위 표 — 상대 끗 0장
- **멍따 (멍텅구리 따블)**: 본인 끗 **7장 이상** (상대 무관)
- **둘 다 만족 시**: ×4

> **현재 코드**: `myYeol >= 5 + 상대 0`만 멍박으로 처리. 멍따 별도 미구현.

> **현재 코드**: 피박이 `myPi >= 10 + 상대 < 5` — 상대 0장 면제 미적용.

## 3. 고 배수 ⭐

| 고 횟수 | 점수 보너스 | 배수 | 누적 결과 |
|---|---|---|---|
| 1고 | +1점 | ×1 | 점수+1 |
| 2고 | +2점 | ×1 | 점수+2 |
| 3고 | +3점 | **×2** | (점수+3)×2 |
| 4고 | +4점 | **×4** | (점수+4)×4 |
| 5고 | +5점 | **×8** | (점수+5)×8 |
| 6고 | +6점 | ×16 | |
| 7고 | +7점 | ×32 | |

> 3고부터 매 고 ×2 누적. **즉 배수 = `goCount >= 3 ? 2^(goCount-2) : 1`**

> **현재 코드**: `if (flags.goCount >= 2) multiplier *= flags.goCount` → ×2/×3/×4. **변경 필요**.

## 4. 흔들기 / 폭탄

| 항목 | 조건 | 효과 |
|---|---|---|
| 흔들기 | 손패에 같은 월 3장 → 공개 선언 | 승리 시 ×2 |
| 흔들기 2번 | 두 월 흔들면 | ×4 (각 ×2 누적) |
| 폭탄 | 손패 같은 월 3장 + 바닥 1장 → 한 번에 4장 먹음 | ×2 + 피 1장씩 + 다음 2턴은 손패 안 내고 더미만 뒤집기 |
| 폭탄 + 흔들기 | 폭탄도 흔들기 효과 | 둘 다 ×2 누적 (×4) |
| 총통 | 시작 시 손패 같은 월 4장 | **즉시 승리** (보통 7점 처리 또는 ×4) |

> **총통은 미구현**. 현재 코드는 손패 4장이어도 일반 진행.

## 5. 특수 매칭 — 피 빼앗기 ✅ 구현됨

매칭 시 발생하는 보너스. **상대로부터 피 가져오기** (3인이면 각 1장씩 = 총 2장).

§1-6 매칭 처리 표(Case 1~6)와 함께 봐야 정확. 각 case의 stealPi:

| 상황 | 정의 (§1-6 case) | 가져오는 피 |
|---|---|---|
| **뻑 (싸기)** | Case 3: 바닥 1 + 손 1 + 더미 같은 월 | 0 (대신 stuck) |
| **자뻑** | Case 6: 본인이 싼 뻑을 본인이 회수 | **2장씩** |
| **상대 뻑 회수** | Case 6: 다른 사람이 싼 뻑을 본인이 회수 | 1장씩 |
| **따닥** | Case 5: 바닥 2 + 손 1 + 더미 같은 월 → 4장 | 1장씩 |
| **쪽** | Case 1: 바닥 0 + 손 1 + 더미 같은 월 → 2장 | 1장씩 |
| **싹쓸이 (쓸)** | 매칭 후 바닥 0장 (마지막 턴 X) | 1장씩 |
| **쪽 + 싹쓸이 동시** | 쪽으로 매칭 후 바닥 0 | **2장씩** (중복) |
| **폭탄** | 같은 월 3장 한 번에 내림 | 1장씩 (호스트 룰로 2장 옵션) |

> 자뻑 owner 추적: `stuckOwners[month] = userId` 매핑으로 자동 판정.
> 마지막 턴 싹쓸이 예외 (`isLastTurn`) 적용됨.

## 6. 첫뻑 / 3뻑 (옵션)

- **첫뻑**: 첫 패 낼 때 뻑 → 보너스 점수 (지역 변형, 보통 +N점). **옵션**.
- **3뻑** (한 사람이 뻑 3번 누적): 그 사람 자동 3점 승리 (rule3). **옵션**.

> 미구현. 옵션으로 추가 가능.

---

## 7. 나가리 / 쇼당 (옵션)

| 항목 | 정의 | 효과 |
|---|---|---|
| **나가리** | 모두 점수 못 내고 더미/손패 다 소진 | **다음 판 점수 ×2** |
| **쇼당** | 상대 둘 다 점수 직전 + 내가 낼 패가 둘 다에 영향 | 본인이 "쇼당!" 외치면 무효(나가리) |

> 둘 다 미구현. 옵션.

---

## 8. 매너 / 부가 룰

- **낙장불입**: 한 번 내린 패 회수 X
- **사자무언**: 죽은 자(광판 사람)는 훈수 금지
- **연사금지**: 4인 게임 두 판 연속 죽지 않음 (매너)

코드 영향 없음. 안내문 정도.

---

## 9. 광팔이 (구현됨)

- 4~5명 입장 시 1~2명이 광 팔고 관전
- 자원 / 호스트 지정 / 마지막 입장자 자동 순
- 광판 사람은 게임 종료까지 spectator (화상/이모지 가능)

# 구현 상태 (2026-05-05)

## ✅ 정확성 fix 완료
- 2인/3인 분배 (10/8 vs 7/6)
- 시작 점수 (2인 7점, 3인 3점)
- 고 배수 (1고+1, 2고+2, 3고+ ×2 누적)
- 피박 0장 면제
- 멍박 / 멍따 별도 처리
- **§1-6 매칭 룰 (Case 1~6 정통)** ⭐ 정통 룰 표 그대로
- 종류 다른 2장 매칭 시 사용자 선택 모달

## ✅ Tier 1 구현 완료
- 고박 (독박)
- 총통 (즉시 승리)
- 나가리 (다음 판 ×2)
- 마지막 턴 싹쓸이 예외
- 자뻑 피 2장 / 일반 회수 1장
- 3뻑 자동 승리 (3점)

## ✅ 옵션 구현 완료
- 첫뻑 시각효과 (점수 보너스 X)
- 9월 열끗 ↔ 쌍피 변환 (멀티 한정)
- 쇼당 선언 (본인 턴 → 즉시 나가리)
- 조커 카드 (RoomRules.jokerCount 0~3장)
- 폭탄 보너스 카드 2장 (사용자 변형 — 강제 2턴 skip 대신)

## ⚪ 미구현 (cosmetic / 의도적)
- 분배 시각화 — 4-3-3 stagger는 적용, 정통 "한 바퀴씩 분배" 시각화는 X
- 첫뻑 점수 보너스 — 친구 MVP 의도적 X (돈 X)
- 매너 룰 (낙장불입/사자무언/연사금지) — 코드 영향 X
