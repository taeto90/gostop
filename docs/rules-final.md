## 0. 게임 시작 / 진행 흐름

### 0-1. 인원별 분배 (총 48장)

| 인원 | 손패 (각자) | 바닥 | 더미 | 검산 |
|---|---|---|---|---|
| **2인** (맞고) | **10장** | **8장** | 20장 | 10×2 + 8 + 20 = 48 ✓ |
| **3인** | **7장** | **6장** | 21장 | 7×3 + 6 + 21 = 48 ✓ |
| **4~5인** (광팔이 후 실제 3인) | 7장 | 6장 | 21장 | (3인 게임과 동일) |

### 0-2. 시작 점수 (난다 기준)

| 인원 | 점수 |
|---|---|
| 2인 (맞고) | **7점** |
| 3인 / 광팔이 후 | **3점** |

### 0-3. 분배 방식

선부터 반시계방향:
- **3인**: 4장씩 한 바퀴 → 바닥 3장 → 다시 3장씩 한 바퀴 → 바닥 3장 (총 손패 7장 / 바닥 6장)
- **2인**: 5장씩 한 바퀴 → 바닥 4장 → 다시 5장씩 한 바퀴 → 바닥 4장 (총 손패 10장 / 바닥 8장)

✅ **구현됨**: `dealNewGame`이 인원수에 따라 분기. 본인 시각화는 `lib/dealingPattern.ts`로 4-3-3 stagger.

### 0-4. 분배 시 reshuffle 케이스

분배 직후 다음 케이스는 **자동으로 deck을 다시 셔플**해서 분배 (게임 진행 불가능 또는 부적절):

| 케이스 | 처리 |
|---|---|
| 같은 월 카드 **4장 모두 바닥**에 깔림 | 재분배 (해당 월 카드를 어떤 player도 소유 X — 게임 진행 무의미) |
| 본인 손패 같은 월 **4장** | **총통** — 즉시 승리 (재분배 X, §6 참고). 단, 친구 협의 룰로 "재분배" 옵션도 있음 |

✅ **구현됨**: `dealNewGame`이 바닥 4장 동월 검사 후 자동 reshuffle (최대 10회). 총통은 별도 처리 (게임 시작 시 즉시 phase='ended').

## 1. 점수 계산 — 베이스

### 1-1. 광 (3장 이상)

| 보유 | 점수 | 비고 |
|---|---|---|
| 3광 | 3점 | 비광 미포함 |
| 비삼광 (3광 중 12월 비광 포함) | **2점** | 비광이 끼면 -1점 |
| 4광 | 4점 | 비광 포함이어도 4점 |
| 5광 | 15점 | 광박 자동 적용 (×2 → 30점) |

### 1-2. 끗 (열끗) (5장 이상)

| 보유 | 점수 |
|---|---|
| 5장 | 1점 |
| 6장 | 2점 |
| 7장 | 3점 + **멍따 ×2** (별도 보너스) |
| 8장 | 4점 + 멍따 |
| 추가 1장당 +1점 |

**고도리** (1·2·8월 새 3장): **+5점** 별도 보너스.

### 1-3. 띠 (5장 이상)

| 보유 | 점수 |
|---|---|
| 5장 | 1점 |
| 6장 | 2점 |
| 추가 1장당 +1점 |

**홍단 / 청단 / 초단 (쿠사)** 각 3장 모음 시 각 **+3점** 별도 보너스.

### 1-4. 피 (10장 가치 이상)

| 보유 | 점수 |
|---|---|
| 10장 | 1점 |
| 11장 | 2점 |
| 추가 1장당 +1점 |

**쌍피**: 한 장 = 2장 가치 (11월 쌍피, 12월 비쌍피).
**조커** (옵션) ✅ 구현됨: **쌍피(2장) 가치 고정**. `RoomRules.jokerCount` 0~3장 셔플 추가. 매칭 X — **보너스피와 동일하게** collected 직행 + 더미 일반패 손패 보충 + 턴 유지. **유일한 차이: 조커는 상대 피 빼앗기 X** (§5-2 비교표 참고).

### 1-5. 9월 열끗 쌍피 변환 (옵션) ✅ 구현됨
9월 열끗(`m09-yeol`)을 끗 자리 또는 쌍피 자리로 **게임 중 실시간 토글**.
끗 획득 순간 `NineYeolPickerModal` 자동 표시 → 끗/쌍피 선택 → `game:toggle-9yeol` emit. 본인 턴 중 점수판에서 다시 클릭해 재변환도 가능 (`Player.flags.nineYeolAsSsangPi`).
`calculateScore({ nineYeolAsSsangPi })` 옵션으로 점수 + 박 판정에 반영.
> `allowGukJoon` 옵션은 `m09-ssangpi`가 별도 쌍피였던 시절의 잔재 — default true로 점수 영향 없음.

### 1-6. 매칭 처리 ⭐ 정통 룰 (Image #20 표) ✅ 구현됨

손패 1장을 내고 더미를 1장 뒤집는 매 턴의 매칭 처리:

| # | 상황 | 결과 (용어) | 설명 |
|---|---|---|---|
| 1 | 바닥 0 + 손 1 + 더미 같은 월 | **쪽** | 바닥엔 없었지만 내가 내고 뒤집은 게 짝. **상대 피 1장씩 뺏어옴** |
| 2 | 바닥 1 + 손 1 + 더미 다른 월 | 그냥 먹기 | 일반 매칭. 손패 + 바닥 1장 = 2장 collected |
| 3 | 바닥 1 + 손 1 + 더미 같은 월 | **뻑 (설사)** | 2장 먹으려다 3장이 겹침. 못 가져오고 바닥에 3장 묶어둠 (stuck) |
| 4 | 바닥 2 + 손 1 + 더미 다른 월 | 그냥 먹기 | 바닥 2장 중 1장만 짝 맞춰 가져옴 (1장은 바닥에 남음). **종류 다르면 사용자 선택** ⭐ |
| 5 | 바닥 2 + 손 1 + 더미 같은 월 | **따닥** | 1장 먹고 더미로 나머지 1장까지 = **총 4장**. **상대 피 1장씩 뺏어옴** |
| 6 | 바닥 3 + 손 1 | **뻑 회수** (먹기) | 묶여있던 3장 + 손패 1장 = **총 4장 한 번에**. 자뻑이면 **2장씩**, 일반 회수면 **1장씩** 뺏어옴 |

**선택 모달 노출 조건** (rules-final.md §1-6 case 4):
- 바닥에 같은 월 카드 2장 + 손패가 같은 월
- 두 바닥 카드의 종류가 **다를 때만** 선택 모달 (피+광 등)
- 같은 종류 (피 2장 등)면 자동 첫 번째 — 결과 동일
- 종류 비교: `kind` + `ddiKind` + `isSsangPi` + `isGoDori` + `isBigwang`
- 손패 단계 / 더미 단계 각각 검사 — 둘 다 해당하면 모달 2번
- 모달 시점: **Phase 3 (더미 비행) 후** 표시 → 선택 후 Phase 4 (점수판 비행)

> ⚠️ **싹쓸이 발생 시**: 위 매칭 후 바닥이 비어버리면 `sweep` flag + 피 1장 추가 (마지막 턴 제외).

> **server protocol**: `game:action play-card { cardId, targetAfterHand?, targetAfterDraw? }` →
> 종류 다른 2장 매칭 시 `{ ok: false, needsSelection: { stage, candidates } }` 응답.
> 클라가 모달 띄우고 선택 후 `targetAfterHand`/`targetAfterDraw` 포함하여 재요청.

## 2. 박 (Bak) — 점수 ×2

피박/광박/멍박은 **승자에게 곱셈** (각각 ×2). 누적 시 ×4, ×8.

| 박 | 발동 조건 | 비고 |
|---|---|---|
| **피박** | 승자가 피로 점수 났음(10장+) AND 패자 피 **1~5장** | 패자 피 **0장 = 면제** ⭐ |
| **광박** | 승자가 광으로 점수 났음(3광+) AND 패자 광 **0장** | |
| **멍박** | 승자가 끗으로 점수 났음(5장+) AND 패자 끗 **0장** | |
| **고박 / 독박** | 고 부른 사람이 다른 사람에게 졌을 때 | 점수 ×2, 본인이 다른 패자 몫까지 부담 |

### 2-1. 멍박 vs 멍따 (별개 개념) ⭐
- **멍박**: 위 표 — 상대 끗 0장
- **멍따 (멍텅구리 따블)**: 본인 끗 **7장 이상** (상대 무관)
- **둘 다 만족 시**: ×4

> **현재 코드** ✅: 멍박(`myYeol >= 5` + 상대 끗 0장) + 멍따(`myYeol >= 7`, 상대 무관, `allowMyungttadak` 옵션) **모두 구현**. 둘 다 만족 시 ×4.

> **현재 코드** ✅: 피박 = `myPi 가치 >= 10` + 상대 피 **1~5장** → 상대 피 **0장이면 면제** (`opponentMinPi >= 1 && <= 5`).

## 3. 고 배수 ⭐

| 고 횟수 | 점수 보너스 | 배수 | 누적 결과 |
|---|---|---|---|
| 1고 | +1점 | ×1 | 점수+1 |
| 2고 | +2점 | ×1 | 점수+2 |
| 3고 | +3점 | **×2** | (점수+3)×2 |
| 4고 | +4점 | **×4** | (점수+4)×4 |
| 5고 | +5점 | **×8** | (점수+5)×8 |
| 6고 | +6점 | ×16 | |
| 7고 | +7점 | ×32 | |

> 3고부터 매 고 ×2 누적. **즉 배수 = `goCount >= 3 ? 2^(goCount-2) : 1`**

### 고 외치는 조건 ✅ 구현됨

| 상황 | 조건 |
|---|---|
| **첫 고** | winScore 도달 (3인 = 3점, 맞고 = 7점) |
| **2고+** | **직전 고 점수보다 1점 이상** 올라야 (`lastGoScore + 1`) |

> 본인 턴 종료 후 점수가 조건을 만족하면 GO/STOP 모달. 점수가 그대로면 모달 X (계속 진행).
> `applyGo`에서 `player.flags.lastGoScore = 현재 점수` 기록 → 다음 고 비교 기준.
> AI는 `shouldAIGo(player, room, difficulty)` 난이도별 자동 판단.

## 4. 흔들기 / 폭탄

### 4-1. 발동 시점

| 항목 | 시점 | 비고 |
|---|---|---|
| **흔들기** | **게임 도중** — 손패에 같은 월 3장 보유 + 그 월 카드 클릭 시 모달 발동 | 사용자가 [O/X] 선택 |
| **폭탄** | 흔들기 [O] 선언 + 바닥에 같은 월 1장 있을 때 → 추가 [폭탄/1장 내기] 선택 | **흔들기 미선언 시 폭탄 불가능** |
| **총통** | 게임 시작 시 손패 같은 월 4장 | **즉시 승리** (선택권 X) |

### 4-2. 손패 3장 시 4가지 경우의 수

손패에 같은 월 카드 3장 보유 상태에서 그 월 카드 클릭 시 모달이 뜬다. 모달 응답 + 바닥 상태에 따라:

| 케이스 | 바닥에 매칭 패 | 흔들기 선언 | 동작 |
|---|---|---|---|
| **① 흔들고 폭탄** (정석) | 있음 (1장) | O | • 3장 한 번에 + 바닥 1장 = **4장 collected**<br>• **점수 ×2** (흔들기만 — 폭탄 자체는 배수 X)<br>• 상대로부터 **피 1장씩** (`bombStealCount` 옵션 1 또는 2)<br>• **폭탄 보너스 카드 2장** 손패 추가 → 폭탄 친 턴은 종료, 다음 본인 턴부터 한 장씩 사용 (매칭 X, 더미만 뒤집기) |
| **② 흔들고 1장 내기** | 없음 | O | • 패 공개 후 **1장만** 냄<br>• **점수 ×2** (흔들기)<br>• 보너스 X<br>• 남은 2장은 그대로 손패에 (사용자 자유) |
| **③ 조용히 1장 내기** | 있음 (1장) | X | • 흔들기 거부 → **1장만 내려놓아 바닥 1장과 일반 매칭(2장 수거)**<br>• **점수 ×2 없음** (흔들기 선언 안 함)<br>• 폭탄 발동 X — 폭탄은 흔들기 선언이 전제<br>• 거의 안 쓰임 (×2 + 폭탄 보너스 포기하는 셈) |
| **④ 조용히 찢어 내기** | 없음 | X | • **1장만** placed<br>• **점수 ×2 없음**<br>• 손에 같은 월 보유 사실 비공개 — 안전한 운영 시 사용 |

### 4-3. 흔들기 모달 UI 흐름

```
[손패 같은 월 3장 보유 상태에서 그 월 카드 클릭]
        ↓
   "흔들기 선언?" 모달
        ↓
   [O] 선언 ────┬── 바닥에 매칭 패 있음 → [폭탄/1장 내기] 모달 → 사용자 선택
                │                                    ├─ 폭탄 → ① (×2 + 보너스 카드 2장)
                │                                    └─ 1장 → ② (×2)
                └── 바닥에 매칭 패 없음 → 자동 ② (×2)
   [X] 거부 ────── 일반 매칭 → ③ 또는 ④ (×2 X)
```

### 4-4. 점수 누적

- 흔들기 1개 월: **×2**
- 흔들기 2개 월: ×2 × ×2 = **×4**
- 흔들기 + 폭탄: **×2** (폭탄 자체는 배수 X — 흔들기만 ×2)
- 흔들기 2개 월 + 폭탄: **×4** (흔들기 두 번만)

> **2026-05-17 개정**: 폭탄 자체는 점수 배수 X. 흔들기 ×2만 누적. 폭탄은 피 빼앗기 + 보너스 카드 효과만.

### 4-5. 총통 (4장)

| 항목 | 동작 |
|---|---|
| 총통 발동 | 게임 시작 시 손패 같은 월 4장 → **즉시 phase='ended' + 승리** |
| 사용자 선택 | **없음** — 즉시 승리 (한게임/넷마블 정통 룰의 [계속하기] 옵션 미구현) |
| 점수 | `nagariMultiplier` 누적 ×2 + 기본 광 점수 + 총통 배수 (구현 별도 검증) |

> **참고**: 4장 흔들기 + 다음 턴 폭탄 (피망 변형 ×4)은 미지원. 4장은 항상 총통 처리.

## 5. 특수 매칭 — 피 빼앗기 ✅ 구현됨

매칭 시 발생하는 보너스. **상대로부터 피 가져오기** (3인이면 각 1장씩 = 총 2장).

§1-6 매칭 처리 표(Case 1~6)와 함께 봐야 정확. 각 case의 stealPi:

| 상황 | 정의 (§1-6 case) | 가져오는 피 |
|---|---|---|
| **뻑 (싸기)** | Case 3: 바닥 1 + 손 1 + 더미 같은 월 | 0 (대신 stuck) |
| **자뻑** | Case 6: 본인이 싼 뻑을 본인이 회수 | **2장씩** |
| **상대 뻑 회수** | Case 6: 다른 사람이 싼 뻑을 본인이 회수 | 1장씩 |
| **따닥** | Case 5: 바닥 2 + 손 1 + 더미 같은 월 → 4장 | 1장씩 |
| **쪽** | Case 1: 바닥 0 + 손 1 + 더미 같은 월 → 2장 | 1장씩 |
| **싹쓸이 (쓸)** | 매칭 후 바닥 0장 (마지막 턴 X) | 1장씩 |
| **쪽 + 싹쓸이 동시** | 쪽으로 매칭 후 바닥 0 | **2장씩** (중복) |
| **폭탄** | 같은 월 3장 한 번에 내림 | 1장씩 (호스트 룰로 2장 옵션) |
| **보너스피** | 투피/쓰리피 (옵션 카드) | 가져간 개수만큼 각 상대로부터 1장씩 (§5-2) |

> 자뻑 owner 추적: `stuckOwners[month] = userId` 매핑으로 자동 판정.
> 마지막 턴 싹쓸이 예외 (`isLastTurn`) 적용됨.

## 5-2. 보너스피 (투피 / 쓰리피) ✅ 구현됨 — 사용자 변형 룰

정규 48장 밖의 **옵션 추가 카드**. 호스트 룰 `bonusPiTwoCount` (0~2장) / `bonusPiThreeCount` (0~1장)로 셔플에 포함.

| 카드 | 피 가치 | 비고 |
|---|---|---|
| **투피** (✌️) | 2장 (쌍피) | `isSsangPi=true` |
| **쓰리피** (🤟) | 3장 | `bonusPiValue=3` |

### 손에서 보너스피를 낼 때 (정통 + 사용자 변형)

```
1. 보너스피 → 딴패(collected) 직행 (바닥 경유 X)
2. 가져간 보너스피 개수만큼 상대 피 빼앗기 (각 상대로부터 1장씩, 없으면 skip)
3. 더미에서 1장 뒤집어 → 내 손패로 (상대는 마스킹으로 그 패 모름)
   · 뒤집은 게 또 보너스피/조커면 → 그것도 딴패로 가져가고 (체인) 다음 장 뒤집기
   · '진짜 일반 패'가 나올 때까지 반복 → 그 일반 패만 손패로
4. 손패 수 유지 (보너스피로 줄어든 만큼 더미 카드로 보충)
5. 턴 유지 — 손패가 보충됐으므로 한 번 더 카드를 냄 (턴 안 넘어감)
```

> **체인 누적 stealPi**: 손 + 더미에서 가져간 보너스피 **총 개수**만큼 상대 피를 빼앗음.
> 예) 손 투피 1장 + 더미 체인 쓰리피 1장 = 보너스피 2개 → 상대 피 2장.

### 뻑에 보너스피가 끼면 (stuck)

- 뻑 발생 턴에 가져간 보너스피는 점수판 X → `stuckBonusPis[month]`에 stuck + stealPi 취소
- 회수자가 뻑 회수 시 stuck 보너스피도 함께 가져감 + 추가 stealPi (`보너스피 수 + 1`)

### 조커와의 차이 (2026-06-03 — 조커도 손패 보충 룰 적용)

| 항목 | 조커 🃏 | 보너스피 ✌️🤟 |
|---|---|---|
| 피 가치 | 쌍피(2) 고정 | 투피=2, 쓰리피=3 |
| 더미 카드 행선지 | **내 손패로 보충** (동일) | **내 손패로 보충** |
| 턴 유지 | ✅ 한 번 더 (동일) | ✅ 한 번 더 |
| 체인 (보너스피/조커 연속 뒤집기) | ✅ (동일) | ✅ |
| **상대 피 빼앗기** | ❌ 없음 | ✅ 가져간 개수만큼 |

> **유일한 차이는 상대 피 빼앗기뿐** — 조커는 `bonusPiCollected`를 안 올려 stealPi=0. 나머지(딴패 직행·손패 보충·턴 유지·체인)는 보너스피와 완전히 동일.
> 구현: `game.ts`의 `if (handCard.isBonusPi || handCard.isJoker)` 분기 — `specials.drawnToHand=true`.
> 클라는 4-phase 대신 전용 무음 시퀀스(`runBonusPiSequence`)로 처리 ('착' 소리 X).
> 서버 턴 유지: `turnFlow.ts` / `aiTurn.ts`의 `playedHandRefill` 판정 (보너스피 OR 조커).
