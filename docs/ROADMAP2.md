```

├── apps/
```

### 모바일/PC 분기
- `COMPACT_BREAKPOINT = 950px` 기준
- 모바일 가로 모드 = 950 미만
- 카드 사이즈 결정 원리는 [`apps/web/docs/layout-sizing.md`](../apps/web/docs/layout-sizing.md) 참고
- ⚠️ 카드 키울 때 `*_MAX_WIDTH` cap만 변경하면 효과 없을 수 있음 — heightBased가 결정자라면 `HAND_AREA_RATIO` 같은 영역 height를 조절해야 함

### 시퀀스 애니메이션 (Phase 1~4)
손패 카드 클릭 시 발생하는 4단계 비행:
- Phase 1: 손패 → 바닥 (확대 + 비행)
- Phase 2: 착지 사운드
- Phase 3: 더미에서 카드 뒤집어 빈 자리로 비행
- Phase 4: 매칭 카드들 한 장씩 점수판으로

타이밍은 `animationTiming.ts`에서 조절. AI는 사용자보다 2배 빠름.

### 솔로 / 멀티 동작
- `/solo` — 1대1 또는 1대2 AI (난이도 3종)
- `/room/:id` — 멀티 (Socket.io, 서버 권위)
- `/result-demo` — 결과 화면 디자인 검토용 (mock data)

### 배포 상태
- **로컬 dev only**. Vercel/Railway 배포 안 됨.
- LiveKit 화상 — mock placeholder 상태, 실제 통합 안 됨.

## B. 추가 추천 기능 (ROADMAP에 없음)

### B.1 ⭐ 텍스트 채팅 (in-game) — Tier 1
**왜 필요**: 화상으로 말하기 어려운 상황(공공장소, 음소거 환경)에서도 친구와 게임 중 빠른 의사소통.

**구현**:
- 우측 토글 버튼 (이모지처럼) → 채팅 모달 펼침
- Socket.io `chat:send` / `chat:broadcast`
- 메시지 메모리 보관 (5분 TTL), 새로고침 시 사라짐
- 욕설 필터 (선택)

**예상**: 1일

---

### B.2 ⭐ 도움말 / 룰 가이드 — Tier 1
**왜 필요**: 친구가 화투를 처음 해보는 경우 로비/게임 중에서 룰 빠르게 확인 가능.

**구현**:
- 로비 우측 상단 "?" 버튼 → 모달
- 카드 분포(48장), 점수 계산(광/끗/띠/피), 박/고스톱, 보너스 룰 (따닥/쪽/뻑) 한 페이지 요약
- 게임 중에도 "?" 버튼으로 빠른 참조

**예상**: 1일 (콘텐츠 작성 + 모달 UI)

---

### B.3 ⭐ 효과음 볼륨 조절 + 음소거 토글 — Tier 1
**현 상태**: `localStorage['gostop:muted']`만 있음. UI는 없음.

**구현**:
- 우측 상단 🔊 / 🔇 토글 버튼
- 0~100 슬라이더 (옵션)
- Howler `Howler.volume(N)` 적용

**예상**: 0.5일

---

### B.4 ⭐ 게임 시간 제한 (각 턴 N초) — Tier 1
**왜 필요**: 친구가 너무 오래 고민하면 분위기 늘어짐. 옵션으로 켤 수 있게.

**구현**:
- 방 만들기 옵션: "턴 제한 30초/60초/없음"
- 턴 시작 시 카운트다운 UI (헤더 옆 또는 손패 위)
- 시간 초과 시 자동으로 매칭 가능한 카드 중 무작위 하나 플레이 (또는 랜덤 카드 깔기)

**예상**: 1일

---

### B.5 ⭐ 게임 히스토리 / 전적 (localStorage) — Tier 1
**현 상태**: `StatsBoard`로 세션(현재 새로고침 전까지) 통계만 있음.

**구현**:
- 게임 종료 시 결과를 `localStorage['gostop:history']`에 저장 (최근 50판)
- 로비/결과 화면에서 "전체 전적" 버튼 → 모달로 표시
- 친구별 승률, 평균 점수, 최고 점수 등

**예상**: 1일

---

### B.6 PWA 지원 (오프라인 설치) — Tier 2
**왜 필요**: 친구들이 모바일에서 "홈 화면에 추가" → 앱처럼 사용 가능.

**구현**:
- `vite-plugin-pwa` 추가
- manifest.json (아이콘, 이름, theme color)
- service worker (정적 파일 캐싱)
- 앱 아이콘 디자인

**예상**: 0.5일

---

### B.7 결과 화면 스크린샷 / 공유 — Tier 2
**왜 필요**: "와 나 이만큼 이김!" 친구에게 자랑 가능.

**구현**:
- 결과 화면 우측 상단 "📷 캡처" 버튼
- `html2canvas` 또는 `dom-to-image`로 결과 화면 캡처
- 다운로드 또는 클립보드 복사

**예상**: 1일

---

### B.8 카드 테마 / 다크-라이트 모드 — Tier 2
**왜 필요**: 사용자 선호도. 야간 다크, 낮 라이트.

**구현**:
- 설정 모달에 "테마: 자동/다크/라이트" 토글
- Tailwind dark mode 활용
- 카드 뒷면 디자인 2~3종 선택

**예상**: 1~2일

---

### B.9 다국어 지원 (한국어/영어) — Tier 2
**왜 필요**: 한국어 모르는 친구 (한국에 거주하는 외국인 친구 등)도 게임 가능.

**구현**:
- `react-i18next` 또는 단순 `messages.ko.ts` / `messages.en.ts`
- 룰 용어(광/끗/띠/피) 영어 번역 (gwang/yeol/ddi/pi 또는 의역)
- UI 모든 텍스트 한/영 분리

**예상**: 2일

---

### B.10 키보드 단축키 — Tier 3
**왜 필요**: PC에서 빠른 진행. 마우스 안 쓰고 키보드만.

**구현**:
- 1~7 숫자 키: 손패 N번째 카드 선택
- Space: 매칭 가능한 첫 카드 자동
- G/S: 고/스톱 결정 (모달 떠 있을 때)
- ESC: 모달 닫기 (이미 적용됨)

**예상**: 0.5일

---

### B.11 AI 분석 / 코칭 모드 — Tier 3
**왜 필요**: 게임 끝나고 "이 수가 잘못됐다" 친구가 알려주는 느낌.

**구현**:
- 게임 끝나면 "이 턴에서 이게 더 나았음" 분석 (간단한 휴리스틱)
- 결과 화면에 "분석 보기" 토글
- 매 턴마다 actionable feedback (예: "이때 9월 광을 잡았으면 광박 시킬 수 있었음")

**예상**: 3~4일 (난이도 높음)

---

### B.12 일시정지 / AFK 처리 — Tier 3
**왜 필요**: 화장실 가거나 잠깐 자리 비울 때.

**구현**:
- 일시정지 버튼 (다른 플레이어 동의 필요)
- 30초+ 응답 없을 시 AFK 표시
- 5분+ AFK면 자동 키킹 옵션

**예상**: 1~2일

---

### B.13 음성 인식 "고!"/"스톱!" — Tier 3
**왜 필요**: 화상으로 즐기는 게임의 재미. 진짜 외치면 인식.

**구현**:
- Web Speech API
- 한국어 음성 인식 → "고", "스톱" 키워드 감지
- 자동으로 GoStopModal 결정 적용
- 정확도 낮으니 옵션으로

**예상**: 2~3일 (정확도 튜닝 필요)

---

### B.14 ⭐ AI 난이도별 캐릭터 + 대화 — Tier 1 (재미 요소)
**왜 필요**: 솔로 모드에서 AI가 무미건조함. 캐릭터성 부여하면 더 재미있게.

**구현**:
- 초급/중급/고급 봇이 각자 닉네임/아바타/성격 (예: 초급 = "김초보", 고급 = "박고수")
- 게임 중 가끔 채팅 (이미 채팅 있다면): "오 좋은 패네", "박 시키지 마...", "고!", "광박이다 ㅋㅋ"
- 결과화면에서 캐릭터 표정 변화 (이김/짐)

**예상**: 1~2일

---

### B.15 ⭐ 화면 캡처 방지 / 스트리머 모드 — Tier 3
**왜 필요**: 트위치/유튜브 방송할 때 다른 플레이어 손패 가리기.

**구현**:
- 옵션: "스트리머 모드 ON" → 본인 손패가 마우스 hover 시만 보임
- OBS 캡처용 alpha 채널 background

**예상**: 1일

---

### B.16 ⭐ 룰 변형 — 데모 모드 / 빨리진행 모드 — Tier 2
**왜 필요**: 친구들이 룰 익힐 때 빠르게 한 판.

**구현**:
- "데모 모드": 손패 5장 (대신 7장), 더미 짧음 → 1판 30초 안에
- "빨리진행 모드": 모든 애니메이션 시간 50% (animationTiming.ts 한 번 더 줄임)

**예상**: 0.5일

---

### B.17 친구 자동 알림 (Web Notification) — Tier 2
**왜 필요**: 멀티 게임 중 다른 탭에 있을 때 본인 차례 되면 알림.

**구현**:
- `Notification.requestPermission()`
- 본인 차례 시작 시 알림 (탭 비활성 상태일 때만)
- 옵션 토글

**예상**: 0.5일

### B.19 ⭐ 1인 자동 AI 모드 (멀티 ↔ 솔로 통합) — Tier 1
**왜 필요**: 친구가 아직 안 모였거나 1명뿐이어도 즉시 게임 가능. 솔로 연습용 + 친구 기다리는 동안 한 판.

**구현**:
1. 멀티 방에 player가 1명일 때도 게임 시작 허용
2. 게임 시작 시 부족한 인원만큼 **AI 봇 자동 추가** (`ai-bot-1`, `ai-bot-2` 등)
   - 1명: AI 1~2 추가 (난이도 옵션)
   - 2명: 그대로 또는 AI 추가
3. 서버측 AI 턴 자동 진행
   - `chooseAiCard`(`packages/shared/src/ai`)를 서버에서 호출
   - AI 차례 되면 setTimeout으로 자연스러운 딜레이 후 액션
4. 친구가 합류 시 처리 정책:
   - 진행 중인 판: 다음 판부터 합류 (관전자로 입장)
   - 또는 즉시 AI를 spectator로 빼고 합류 player로 (옵션)
5. UI:
   - 1인 모드 진입 시 "AI와 대전" 안내
   - 게임 화면에서 AI 닉네임/난이도 표시 (현재 SoloPlay와 통일)

**의존성**:
- 현재 `SoloPlay`(`apps/web/src/features/solo`)의 AI 진행 로직을 서버로 이식
- 서버 setTimeout으로 봇 턴 진행 + 모든 클라에 broadcast
- 솔로/멀티 코드 통합 — 솔로 모드 라우트는 유지하되 내부적으로 1인 멀티 방으로 redirect 가능

**예상**: 3~5일 (큰 작업, 솔로/서버 로직 통합 + AI 서버 진행)

**우선순위**: ⭐ Tier 1 — 친구 안 모여도 즉시 즐길 수 있음. 진입 장벽 낮춤.

### B.18 게임 룰 변형 토글 (Phase 4 Tier 2 재정리) — Tier 2
- 시작 점수 (3점/5점/7점)
- 국준 인정 여부
- 흔들기 처리 (×2 / 점수만 추가)
- 멍따 인정 여부
- 폭탄 시 피 빼앗기 갯수 (1장/2장)

방 만들기 시 옵션, 게임 중에는 변경 불가. ※ **B.18은 H. 방장 기능과 연계됨** — 방장만 변경 가능.

**예상**: 1~2일

## H. 🏠 방장(Host) 기능 — Tier 1 (큰 작업)

### 배경
현재 멀티 모드(`apps/server/src/socket/handlers.ts`)는 방을 만든 사람을 `hostUserId`로
저장하긴 하지만, 클라이언트에서 그 권한을 활용하지 않음. 룰 변경, 강퇴, 게임 시작
모두 권한 구분 없이 누구나 가능한 상태로 추정.

이 섹션은 **방장 권한 시스템 + 관련 부가 기능**을 정리합니다.

---

### H.1 ⭐⭐ 방장 권한 시스템 — Tier 1
**구현**:
1. 서버: `Room.hostUserId` 이미 있음. 모든 권한 액션은 `socket.userId === room.hostUserId` 검증
2. 액션 화이트리스트 (방장만):
   - 게임 시작 (`game:start`)
   - 룰 변경 (`room:update-rules`)
   - 다른 플레이어 강퇴 (`room:kick`)
   - 방장 위임 (`room:transfer-host`)
   - 다음 판 시작 (`game:next-round`)
3. 클라이언트: `myUserId === view.hostUserId` 일 때만 권한 UI 표시 (👑 아이콘 등)
4. 방장이 떠나면 **자동 위임** (다음 플레이어 또는 가장 오래된 멤버)

**예상**: 1일

---

### H.2 ⭐ 방 룰 설정 모달 (방장 전용) — Tier 1
B.18 룰 변형을 방장만 변경 가능하게:
- 방 만들기 화면에서 초기 룰 설정
- 대기실(WaitingRoom)에서 방장이 ⚙️ 버튼 → 룰 변경 모달
- 게임 진행 중에는 잠금 (수정 불가, 다음 판부터 적용)
- 룰 변경 시 다른 플레이어에게 broadcast (예: "방장이 시작 점수를 5점 → 7점으로 바꿨습니다")

**룰 항목**:
- 시작 점수 (3 / 5 / 7점)
- 흔들기 처리 (×2 점수 / 점수 +α)
- 폭탄 피 빼앗기 (1장 / 2장)
- 국준 인정 여부
- 멍따 인정 여부
- 턴 시간 제한 (없음 / 30초 / 60초)
- 점수 7점 도달 시 자동 STOP / 모달 띄움

**구현**:
- `RoomRules` 타입을 `packages/shared`에 정의
- `Room.rules: RoomRules` 추가
- 게임 로직(`executeTurn`, scoring)이 `rules` 인자 받음

**예상**: 2일

---

### H.3 ⭐ 강퇴 / 방장 위임 — Tier 1
**구현**:
- WaitingRoom에서 방장이 다른 플레이어 옆에 "..." 메뉴 → "강퇴" / "방장 위임"
- 강퇴 시 해당 플레이어 socket disconnect + room에서 제거 + 다른 멤버에게 알림
- 방장 위임 시 `hostUserId` 변경 + 새 방장에게 권한 표시

**예상**: 1일

---

### H.4 ⭐ 게임 시작 / 다음 판 / 종료 결정 (방장 전용) — Tier 1
**현 상태**: 게임 시작은 누구나 가능한 듯 (확인 필요).

**구현**:
- WaitingRoom에서 "게임 시작" 버튼은 방장만 활성화
- 게임 끝나면 "다음 판" 버튼은 방장만 (다른 멤버는 "준비 완료" 상태 표시)
- 모든 멤버가 "나가기" 또는 5분 idle 시 자동 종료

**예상**: 0.5일

---

### H.5 비공개 방 / 비밀번호 — Tier 2
**왜 필요**: 정확히 친구들과만 게임. URL 노출되어도 비밀번호 없으면 입장 불가.

**구현**:
- 방 만들기 옵션: "공개 / 친구 초대만 (URL 비밀번호 4자리)"
- 비공개 방은 방 ID + 4자리 비밀번호 모두 있어야 입장
- 비밀번호 자동 생성 (4자리 숫자) 또는 방장 직접 입력

**예상**: 1일

---

### H.6 방 잠금 (게임 시작 후 입장 차단) — Tier 1
**왜 필요**: 게임 진행 중 모르는 사람 들어와서 방해 방지.

**구현**:
- 게임 시작과 동시에 `room.status = 'in-game'`
- in-game 방에 join 시도하면 reject (`"이미 게임 중입니다"`)
- 단, 관전자(`asSpectator: true`)는 입장 가능 (옵션)

**예상**: 0.5일

---

### H.7 방 설정 — 인원 수 / 관전자 허용 — Tier 1
**구현**:
- 방장이 룰 모달에서 함께 설정
- 최대 플레이어 (2 / 3 / 5)
- 관전자 허용 (예/아니오)
- 관전자 화상채팅 참여 (예/아니오 — Phase 3에서)

**예상**: 0.5일 (B.18, H.2와 통합)

---

### H.8 방장 표시 시각 효과 — Tier 1
**구현**:
- 방장 닉네임 옆 👑 아이콘
- 방장 본인은 점수판/헤더에 "당신은 방장입니다" 작은 표시
- WaitingRoom에서 방장 슬롯 강조 (테두리/배경)

**예상**: 0.3일

---

### H.9 일시정지 / 멈춤 (방장 전용) — Tier 2
**왜 필요**: 친구가 잠깐 자리 비울 때.

**구현**:
- 방장이 "⏸️ 일시정지" 버튼 → 모든 게임 액션 잠금
- 다른 플레이어 화면에 "방장이 잠시 자리를 비웠습니다" 오버레이
- 방장이 "▶️ 재개" 누르면 진행
- 대안: 일시정지 권한을 모든 플레이어에 (각자 1회/판 한정)

**예상**: 1일 (B.12 일시정지와 통합 가능)

---

### H.10 방 정보 공유 (URL + QR) — Tier 1
**현 상태**: 친구 초대 링크 / QR 미확인. ROADMAP A.2 작업과 연계.

**구현**:
- WaitingRoom에 "친구 초대" 버튼 → 모달
- URL 복사 버튼 (`https://gostop.app/room/ABC123`)
- QR 코드 (`qrcode` lib)
- "카카오톡으로 공유" / "Web Share API" (모바일)
- 비공개 방이면 비밀번호도 함께 표시

**예상**: 0.5일

---

### H.11 방 종료 시 결과 자동 집계 — Tier 2
**왜 필요**: 여러 판 한 후 누가 1등인지.

**구현**:
- 누적 점수는 이미 `StatsBoard`에서 처리됨
- 방장이 "방 종료" 누르면 최종 결과 화면 → 모든 멤버에게 표시
- 스크린샷 공유 가능 (B.7)

**예상**: 0.5일

---

### H.12 추천 방장 부가 기능 (Tier 2/3)
- **자동 매칭 시작 카운트다운** — "5, 4, 3...!" 시작
- **방장 의도적 룰 빌드** — 룰 프리셋 ("정통 / 빠른 진행 / 입문자용") 저장
- **방 채널** — 같은 방장이 만든 여러 방 관리 (게임방 + 음성방)
- **방장 인계 시 인사 메시지** — 방장 변경 시 자동 채팅 알림
- **방장 게임 통계** — "이 방에서 N판 진행 / 평균 점수 / 우승 횟수"
